# CareerPilot

Project context for Claude Code. Read these specs before working on any feature.

## Canonical specs
- `docs/kickoff.md` — what to build: phase scope, Prisma schema, API contracts, Claude API system prompts
- `docs/division-of-powers.md` — operating model: what Claude Code owns vs what Claude.ai owns

## Tone constraint (non-negotiable)
The product addresses users who may be anxious about AI replacing their jobs. All user-facing copy and Claude API output must be **objective, constructive, never alarmist**. Pair exposed-task content with defensible-task framing. Frame exposure as task-level, not person-level. See `docs/kickoff.md` §6.

## Phase scope
Currently **Phase 1 MVP** — assess + plan only. Do not build: Stripe (Phase 3), Expo mobile (Phase 2), email (Phase 3), content feed (Phase 3), trend charts (Phase 3). See `docs/kickoff.md` §10–12.

## Push discipline
Repo is private. Only Claude Code pushes. Spec drafts come from Claude.ai — do not draft new specs here. See `docs/division-of-powers.md`.
